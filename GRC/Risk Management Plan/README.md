# Risk Management Plan — Student Online Marketplace

**Type:** Written project deliverable

## Purpose

A concise risk management plan for a student online-marketplace project, identifying the risks that could derail it and defining practical mitigation and monitoring strategies.

## Key topics

- **Four risk categories** — Technical (bugs, downtime, payment failures), Security (data breach, unauthorized access, payment fraud), Financial (overspending), and Operational (delays, unclear communication), each rated by impact.
- **Mitigation strategies** — concrete, budget-aware controls per category: weekly feature testing and monitoring, MFA and regular vulnerability scans, expense tracking, and clear phase ownership.
- **Risk monitoring** — reviewing risks at the end of each project phase, updating mitigations as the project evolves, and documenting new risks found during testing or launch.

Security is treated as the highest-impact category, with MFA, vulnerability scanning, and least-privilege admin access as the primary controls.
