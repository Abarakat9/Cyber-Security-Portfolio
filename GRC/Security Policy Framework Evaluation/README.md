# Security Policy Framework Evaluation

**Type:** Hands-on lab report (Access Control & Identity Management, Lab 10) · **Standards:** NIST password guidance, CIS-style configuration standards

## Purpose

Evaluate a real security policy document against NIST best practices, recommend revisions to bring it into alignment, and classify a configuration standard's recommendations as satisfied or violated.

## Key topics

- **Policy evaluation** — assessed a password policy against NIST best practices and found it failing on modern standards (no breach checks, no complexity rules), with specific recommended revisions (e.g., rotation and password-history requirements).
- **Policy-framework classification** — determined the document is best described as a *password policy* within the broader policy framework, and why enforcement matters.
- **Configuration-standard review** — examined how a security configuration standard achieves community consensus, and classified five best practices as satisfied / violated / not addressed.
- **Challenge exercise** — compared a real Cisco VPN configuration standard against NIST SP 800 remote-access / BYOD guidance, concluding it partially satisfies best practice and would fully meet it with MFA added.
