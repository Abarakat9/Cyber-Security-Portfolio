# Remote Access and Physical Security Plan — "Big Tire"

**Type:** Group project (ITEC 312) · **My contribution:** the cost-effective MFA plan + final submission

## Purpose

A security plan for a fictional multi-location company ("Big Tire") covering three areas: secure remote access to data-center applications, physical security across sites, and a cost-effective multi-factor authentication rollout for web systems.

## Key topics

- **Remote access** — Virtual Desktop Infrastructure (VDI), secure HTTPS access, MFA, a remote-access gateway, and performance optimization (load balancing, application acceleration).
- **Physical security** — smart-card access control, programmable locks, CCTV surveillance, employee training, remote monitoring/alarms, and scheduled access audits.
- **Cost-effective MFA (my section)** — a tiered authentication plan (free authenticator apps for most users, YubiKey hardware keys for high-risk users at $60/unit, built-in biometrics) with a per-location budget breakdown across five sites (~$100K–150K annually) and a justification balancing security against cost and user convenience.

**Standards referenced:** NIST SP 800-63-3 (Digital Identity Guidelines), plus vendor guidance from Cisco, Microsoft, and Duo.
